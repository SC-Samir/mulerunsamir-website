Automatiser vos déploiements Scalingo avec MuleRun

Un guide étape par étape pour orchestrer votre DevOps en langage naturel.

Ce tutoriel démontre comment utiliser MuleRun, votre agent de développement propulsé par l'IA, pour automatiser entièrement la création, la configuration et le déploiement d'une application Node.js avec mise en cache Redis sur le PaaS Scalingo.

1. Ce que vous allez construire

Vous allez demander à MuleRun de construire une application web Node.js haute performance qui :

  • Se connecte à une base de données Redis managée pour la mise en cache et la gestion d'état.
  • Dispose d'une interface utilisateur moderne et responsive.
  • Est déployée automatiquement sur Scalingo via l'intégration GitHub.
  • Est entièrement vérifiée par MuleRun grâce à l'analyse automatique des logs et des requêtes HTTP.

2. Prérequis

  • Un accès à MuleRun.
  • Un token API Scalingo (Générez-en un sur : https://dashboard.scalingo.com/account/tokens).
  • Un Personal Access Token GitHub avec les scopes repo et read:org.

Note : Vous n'avez pas besoin d'installer Node.js, Git ou la CLI Scalingo sur votre machine locale. MuleRun gère l'environnement pour vous.

3. Étape 1 : Générer l'application avec MuleRun

Copiez et collez ce prompt dans MuleRun :

"Crée une application web Node.js dans un nouveau dossier appelé `mulerun-scalingo-app`. Elle doit inclure : 1. Un `package.json` avec `redis` comme dépendance et Node.js `20.x`. 2. Un fichier `server.js` qui se connecte à Redis via `SCALINGO_REDIS_URL` avec un mécanisme de cache (TTL 10s). 3. Un `Procfile` avec la commande `web: node server.js`. 4. Un fichier `public/index.html` moderne avec un thème sombre."

Ce que fait MuleRun : Analyse la demande, génère un code prêt pour la production, crée la structure des dossiers et garantit les bonnes pratiques.

4. Étape 2 : Automatiser la configuration Scalingo

"J'ai mon token API Scalingo : `tk-us-VOTRE_TOKEN`. Veuillez utiliser la CLI ou l'API Scalingo pour : 1. Créer une nouvelle application nommée `my-mulerun-app`. 2. Provisionner un addon `redis-starter-256`. 3. Confirmer que l'application et l'addon ont été créés avec succès."

Ce que fait MuleRun : S'authentifie, exécute les commandes dans son sandbox sécurisé, et vous confirme le succès avec l'URL de l'application.

5. Étape 3 : Déclencher le déploiement avec GitHub

"J'ai un token GitHub : `ghp_VOTRE_TOKEN`. Veuillez : 1. Initialiser un dépôt Git et créer un dépôt GitHub public. 2. Commiter le code et le pousser sur la branche `main`. 3. Lier ce dépôt à mon application Scalingo via l'intégration GitHub. 4. Déclencher un déploiement manuel."

6. Étape 4 : Vérifier et itérer

"Vérifie les logs de déploiement pour t'assurer que le build a réussi et que le client Redis s'est connecté. Ensuite, fais une requête HTTP vers l'URL de l'application et montre-moi la réponse."

En cas d'erreur, MuleRun lira automatiquement les logs, diagnostiquera le problème, corrigera le code et relancera le déploiement.

7. Pourquoi utiliser MuleRun avec Scalingo ?

Flux de travail traditionnel Flux de travail automatisé avec MuleRun
Écriture manuelle du code répétitif. Génération instantanée de code prêt pour la production.
Installation locale des outils (Git, CLI, Node). Zéro configuration locale. Sandbox sécurisé et éphémère.
Copier-coller des commandes CLI et débogage. Orchestration en langage naturel.
Vérification manuelle des logs et tests. Vérification automatisée et preuve du succès.
Correction des erreurs nécessitant un changement de contexte. Auto-réparation (Self-healing) en boucle fermée.

8. Pourquoi Scalingo est la plateforme idéale pour les agents IA ?

  • Approche "API-First" : Scalingo expose une API REST complète et une CLI robuste pour un contrôle 100% programmatique.
  • Configuration automatique : L'injection automatique des variables d'environnement (ex: SCALINGO_REDIS_URL) simplifie la gestion des secrets pour l'agent.
  • Environnement prévisible : Les buildpacks standardisés éliminent les erreurs "ça marche sur ma machine".
  • Boucle de rétroaction rapide : L'accès instantané aux logs permet à l'IA de diagnostiquer et d'auto-corriger les erreurs en quelques secondes.
  • Souveraineté et sécurité : L'hébergement européen (3DS Outscale) et les certifications comme SecNumCloud garantissent la conformité des projets.
